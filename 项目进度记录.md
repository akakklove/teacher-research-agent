## 第一部分：项目介绍

### 1.1 背景与问题

高校教师的科研数据分散在多个系统（科研管理系统、财务系统、人事系统）。教师想了解自己的科研全貌——主持了多少项目、经费到账了多少、发表了几篇论文、获得了什么奖项——往往需要：

1. 登录不同系统逐个查询
2. 手工整理成 Excel 或 PPT
3. 遇到系统间数据不一致时反复核对

一次年度总结或职称评审材料准备，耗时数天。

### 1.2 产品定位

**一句话**：教师登录后，用自然语言问一问，系统自动生成一份个人科研全景大屏报告。

**目标用户**：
- 一线教师（查个人科研数据、做年度总结、准备职称评审材料）
- 院系领导（查看本院教师科研产出对比）
- 科研管理部门（全校科研数据总览）

**核心价值**：把"登录 5 个系统 + 手工整理 3 天"压到"说一句话 + 等 3 秒"。

### 1.3 产品形态

```
教师端：
├── 登录（工号 + 密码）
├── 自然语言对话（"我近三年的科研情况怎么样"）
├── 场景模板（年度总结 / 职称评审 / 基金申报支撑）
├── 指标超市（手动勾选想要的指标）
├── 大屏预览（ECharts 图表 + AI 洞察文字）
├── 导出功能（PPT / PDF / Word 表格）
└── 模板保存与复用

超管端（V1.0）：
├── 数据源配置（MySQL / Oracle / CSV）
├── 字段映射配置（源字段 → 标准字段的 ETL 映射）
├── 定时同步调度 + 日志监控
└── 用户管理
```

### 1.4 技术架构（五层）

```
┌─────────────────────────────────────────────┐
│  展示层    HTML大屏 + Chat UI + 指标选择器     │  ← V0.3
├─────────────────────────────────────────────┤
│  Agent层   意图路由 + 指标引擎 + 洞察 + 编排   │  ← V0.2
├─────────────────────────────────────────────┤
│  服务层    FastAPI + 数据接口 + 安全校验      │  ← V0.2
├─────────────────────────────────────────────┤
│  数据层    MySQL + DataSource Interface     │  ← ✅ V0.1
├─────────────────────────────────────────────┤
│  治理层    Schema Registry + Mock Generator  │  ← ✅ V0.1
└─────────────────────────────────────────────┘
```

**关键设计决策**：19 张应用层表 = 固定数据契约。源系统（科研管理系统、财务系统）的变化通过 ETL 映射层隔离，Agent 层和展示层完全不感知源系统长什么样。

### 1.5 数据模型

以 `t_ky_xmry`（项目人员关系表）为核心枢纽表：

```
t_jzg_jbxx (教师) ──→ t_ky_xmry (项目人员) ←── t_ky_xmjbxx (项目)
                           │
          ┌────────────────┼────────────────┐
          ↓                ↓                ↓
    经费到账/支出/外拨    成果(论文/专利/   活动(会议/机构)
                          著作/软著/获奖)
```

所有以教师为维度的查询，都从 `t_ky_xmry` 出发，通过 `kybh`（项目编号）关联到项目的经费、成果、活动数据。

### 2.1 已完成 — 数据治理层

**Schema 注册中心**：
- 从高校科研管理数据标准 Excel 解析出全部 15 个数据集、279 张表、3,636 个字段
- 每个数据集独立存储为 JSON，包含完整字段注释、类型、主键、非空约束、业务描述
- 自动检测出 12 条跨表关联关系

**Mock 数据生成器**：
- 50 名教师、200 个项目、300 篇论文、80 项专利、40 本著作、50 项软著、60 项获奖、30 场学术会议、15 个科研机构
- 总计 4,252 条数据，覆盖 19 张表的完整关联
- 固定随机种子（SEED=42），每次运行生成完全相同的数据，方便调试
- 内置业务特征规则：教授更容易拿国家级项目、计算机学院论文产出高于文科学院、经费按学院系数加权

**30 个核心指标**：

| 模块 | 指标示例 |
|------|----------|
| 项目概览（5个） | 主持项目数、项目级别分布、项目来源分布、项目状态分布、年度立项趋势 |
| 经费分析（6个） | 到账经费总额、月度到账趋势、经费执行率、支出结构、年均经费对比 |
| 论文成果（5个） | 发表论文数、论文级别分布（SCI/EI/核心）、年度论文趋势、第一作者排名 |
| 专利成果（3个） | 专利总数、专利类型分布、年度申请趋势 |
| 著作/软著（4个） | 著作数、软著数、出版社分布、年度趋势 |
| 获奖荣誉（3个） | 获奖总数、获奖级别分布、获奖类别分布 |
| 学术活动（2个） | 学术会议参与次数、特邀报告次数 |
| 对比分析（2个） | 院系均值对比、同职称教师排名 |

**验收标准**：

**交互流程**：

### B. 设计备忘

- 表名全部小写，MySQL 8.0 兼容
- 编码全链路 utf8mb4，已解决 Navicat 乱码（需 `skip-character-set-client-handshake`）
- 核心枢纽表 `t_ky_xmry`，通过 `ry_gh` 关联教师，通过 `kybh` 关联所有项目/成果
- Mock 数据固定种子（SEED=42）保证可复现
- 19 张表 = 应用层固定 Schema，源系统变化通过 ETL 映射隔离
- 产品话语中统一使用"高校科研管理数据标准"
