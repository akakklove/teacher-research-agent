## 一、项目定位

| 维度 | 内容 |
|------|------|
| 产品名 | 教师个人科研小助手 |
| 核心功能 | 教师登录 → 自然语言提问 → 生成个人科研全景大屏报告 |
| 目标用户 | 高校教师（查个人科研数据、做年度总结、准备职称评审材料） |
| 技术栈 | Python 3.13 + FastAPI + MySQL 8.0 (Docker) + ECharts + LLM |

---

## 二、技术架构

```
展示层 (HTML大屏 + Chat UI)    ← 待做
Agent层 (指标引擎 + 洞察 + 编排) ← 待做
服务层 (FastAPI + 数据接口)     ← 待做
数据层 (MySQL + ETL)           ← ✅ 已完成
```

**关键设计决策**：19张表 = 应用层固定 Schema，源系统变化通过 ETL 映射隔离。Agent 层只认这 19 张表，不感知源系统长什么样。


```
    │ gh = ry_gh

    │ kybh = kybh
```

## 六、核心 SQL 示例

### 查询教师主持的项目
```sql
SELECT p.kybh, p.xmmc, p.xmjb_bm, p.lxrq, p.ht_jf, p.zxzt_bm
FROM t_ky_xmry r
JOIN t_ky_xmjbxx p ON r.kybh = p.kybh
WHERE r.ry_gh = 'GH20200001' AND r.smsx = 1;
```

### 查询教师经费到账总额
```sql
SELECT COALESCE(SUM(f.dz_je), 0) AS total_fund
FROM t_ky_xmry r
JOIN t_ky_jfdz f ON r.kybh = f.kybh
WHERE r.ry_gh = 'GH20200001' AND r.smsx = 1;
```

### 查询教师第一作者论文
```sql
SELECT p.lwmc, p.kwmc, p.kwjb, p.fbn
FROM t_ky_lwzz a
JOIN t_ky_lw p ON a.lwbh = p.lwbh
WHERE a.zzgh = 'GH20200001' AND a.smsx = 1;
```

| 文件 | 说明 |
|------|------|
| `data_governance/mock_generator/generator.py` | 数据生成引擎 |
| `data_governance/mock_generator/seeder.py` | MySQL 灌入脚本 |
| 随机种子 | `SEED=42`（固定可复现） |

**业务特征规则**：
- 教授 25% / 副教授 35% / 讲师 30% / 助教 10%
- 教授更容易拿到国家级项目，讲师多是企业委托
- 计算机学院、生物学院论文产出高于数学学院、法学院
- 项目经费 = 基础范围 × 学院经费系数（计算机 3.0 > 法学 0.2）

## 十、下一步计划

### Phase 1：指标层
- `metrics.yaml` — 30 个指标定义 + SQL 模板
- `Metric Engine` — 指标查询引擎（输入教师工号 → 返回所有指标值）
- `Query Builder` — 参数化 SQL 生成 + 安全校验

### Phase 2：Agent 层
- `Intent Router` — 意图识别（"我的科研情况"→ 推荐指标组合）
- `Insight Engine` — 数据 → 自然语言洞察（趋势/对比/建议）
- `Dashboard Composer` — 指标列表 → ECharts 大屏配置 + HTML 渲染

### Phase 3：展示层
- ECharts 大屏 HTML 模板（KPI卡片 + 折线/柱状/饼图 + 洞察卡片）
- FastAPI 服务 + 教师登录（JWT）
- 场景模板引擎（年度总结 / 职称评审）

### Phase 4（未来）：超管端
- ETL 字段映射配置
- 数据源管理
- 定时调度 + 同步日志

## 十一、备忘

- **表名全部小写**，MySQL 8.0 兼容

- **核心枢纽表** `t_ky_xmry`，通过 `ry_gh` 关联教师，通过 `kybh` 关联所有项目/成果
